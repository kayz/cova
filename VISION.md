# COVA Vision (v2)

## 一句话愿景

让 `coco`、`openclaw` 等任何助理 Agent，都能稳定调用外置专家网络完成复杂工作，而不是在主会话里硬撑复杂流程。

## 背景与问题

我们已经有个人助理 Agent（`coco`），但在真实工作中，复杂任务通常具备以下特征：

- 多步推理与流程控制，不适合只靠提示词串联。
- 需要专家私有知识、私有数据源、专属工具链。
- 运行耗时长、成本高、结果稳定性要求高。

如果全部放在助理 Agent 的 subagent/skills 内执行，会出现两个核心问题：

- token 消耗过高，复杂任务成本难控。
- 结果波动较大，难以形成企业级可预测交付。

## 我们的定位

COVA 是“专家生产力平台”，不是另一个助理 Agent。

- 助理 Agent（如 `coco`）负责：理解意图、拆分任务、发起调用、整合结果。
- COVA 负责：专家注册与治理、任务编排、异步执行、证据化交付、审计与成本管理。
- 专家 Agent 负责：领域决策、流程执行、工具调用、专有数据处理。

## 核心目标

构建一个客户端无关的平台，让调用方只需对接一次接口，即可获得：

- 可预测的复杂任务交付（状态机 + 重试 + 超时 + 幂等）。
- 可验证的输出（answer + confidence + references + freshness）。
- 可运营的专家体系（版本、权限、SLA、成本、审计）。
- 可扩展的生态接入（`coco`、`openclaw`、未来第三方）。

## 设计原则

- Interface First：先冻结契约，再演进实现。
- Multi-tenant by Design：从第一天支持租户与项目隔离。
- Async by Default：复杂任务默认异步执行。
- Evidence over Opinion：关键结论必须有引用证据。
- Runtime Separation：助理与专家运行时解耦，避免主会话过载。
- Operability First：可观测、可回放、可审计、可治理。

## 北极星指标

- 复杂任务成功率（系统侧）>= 99%
- 结果可追溯率（含 references/freshness）= 100%
- 同类任务 token 成本较“助理内联执行”下降 >= 40%
- 新专家接入周期 <= 2 天（含验收）
- 多客户端接入成本 <= 1 个适配层

## 阶段目标

### Phase 1: 平台边界冻结

- 明确双 API（助理侧、专家侧）
- 明确多租户与权限模型
- 明确专家准入规范（Conformance）

### Phase 2: 生产级核心能力

- 可靠队列与可恢复状态机
- 专家生命周期管理（发布/回滚/停用）
- 成本与质量可观测看板

### Phase 3: 生态化扩展

- 支持多个助理 Agent 统一接入
- 构建专家市场化目录与评分体系
- 形成组织级知识与流程复用网络

## 非目标（当前阶段）

- 不追求单一专家“内容最强”。
- 不在助理侧实现复杂重流程。
- 不提前做复杂计费产品化界面。

## 最终图景

COVA 成为 Agent 时代的专家基础设施层：  
助理负责交互，专家负责执行，平台负责可靠交付与治理。
